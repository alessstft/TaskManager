### Task Manager

**Обзор проекта**  
Task Manager - это приложение для мониторинга системных ресурсов Windows, состоящее из двух основных компонентов:

Rust, который помог создать нам DLL для сбора системной информации
Python(tkinter), который создает GUI для отображения данных

**Функциональные возможности системы:**

1. **Вывод данных:**  
    Система выводит данные о компьютере (процессы, службы, производительность) в окна в графическом интерфейсе.
    
2. **Завершение процесса:**  
    Реализована возможность убить процесс и остановить его выполнение.
    
3. **Детализованные данные:**  
    Реализована возможность нажать на кнопки в окне "Производительность", чтобы получить более подробные данные о системе.

## Описание зависимостей

1. **Python 3.9+**
    - Основной язык разработки. Версия 3.9 или выше обеспечивает поддержку всех необходимых функций и библиотек.
    
2. **Windows Api 0.3**
    - Набор базовых функций интерфейсов программирование приложений операционных систем семейств Microsoft Windows. 
    - Предоставляет прямой способ взаимодействия приложений пользователя с операционной системой _Windows_.
    
3. **sysinfo 0.29**
    - Rонтейнер, используемый для получения информации о системе.

4. **Serde 1.9:**
    - Фреймворк для эффективной и универсальной сериализации и десериализации структур данных Rust.

**5. serde_json 1.0**  
- Высокопроизводительная библиотека для работы с JSON в Rust.  
- Обеспечивает парсинг и генерацию JSON-данных.  
- Используется для сериализации и десериализации данных о процессах и системной информации.

**6. lazy_static 1.4**  
- Макрос для Rust, позволяющий инициализировать статические переменные с ленивой инициализацией.  

**7. libc 0.2**  
- Rust-биндинги к библиотеке libc, предоставляющие низкоуровневый интерфейс к API операционной системы.  

**8. num_cpus 1.15**  
- Простая библиотека для определения количества CPU в системе.  
- Предоставляет информацию о количестве физических и логических процессоров.  

**9. ctypes**  
- Стандартная библиотека Python для работы с C-совместимыми типами данных.  
- Позволяет вызывать функции из динамических библиотек и взаимодействовать с кодом на C.  

**10. tkinter**  
- Стандартная графическая библиотека Python для создания кроссплатформенных GUI-приложений.  
- Используется как основа графического интерфейса Task Manager.

###  Модуль структуры
Созданы структуры для передачи данных о системе, сейчас эти структуры будут описаваться.

####  class ProcessInfo(Structure)
- pid - c-строка, которая обозначает ID процесса
- name - c-строка, которая обозначает имя процесса
- cpu_usage - дробное число, которое обозначает, какой процент мощности ЦП использует процесс
- memory_mb - целое число без знака (64 бита), которое обозначает объем памяти, который процесс использует от ОП
- read_kb - целое число без знака (64 бита), которое обозначает сколько памяти процесс требует от диска для чтение
- written_kb - целое число без знака (64 бита), которое обозначает сколько памяти проецесс требует от диска для чтение

####  class ProcessInfoArray(Structure)
- data - указатель на массив структур ProcessInfo
- len - размер массива (количество элементов)

####  class ServiceInfo(Structure)
- process_id - целое число без знака (32 бита), обозначающее ID процесса сервиса
- name - c-строка, содержащая название сервиса
- status - c-строка, содержащая статус сервиса

####  class ServiceInfoArray(Structure)
- data - указатель на массив структур ServiceInfo
- len - размер массива (количество элементов)

####  class CpuStaticInfo(Structure)
- brand - c-строка, содержащая название процессора
- usage - дробное число (float), обозначающее текущую загрузку процессора в процентах
- frequency - дробное число (double), обозначающее текущую частоту процессора
- core_count - целое число, обозначающее количество ядер процессора
- work_time - целое число, обозначающее время работы процессора
- process - целое число (предположительно количество процессов)

####  class MemoryStaticInfo(Structure)
- total - целое число без знака (64 бита), обозначающее общий объем памяти в байтах
- used - целое число без знака (64 бита), обозначающее используемый объем памяти в байтах
- available - целое число без знака (64 бита), обозначающее доступный объем памяти в байтах
- speed - целое число без знака (64 бита), обозначающее скорость памяти
- format - c-строка, содержащая формат/тип памяти

####  class DiskStaticInfo(Structure)
- name - c-строка, содержащая название диска
- total_space - целое число без знака (64 бита), обозначающее общее пространство на диске в байтах
- available_space - целое число без знака (64 бита), обозначающее доступное пространство на диске в байтах

####  class DiskStaticInfoArray(Structure)
- data - указатель на массив структур DiskStaticInfo
- len - размер массива (количество элементов)

####  class NetworksStaticInfo(Structure)
name - c-строка, содержащая название сетевого интерфейса
ipv4 - c-строка, содержащая IPv4 адрес интерфейса
send - целое число без знака (64 бита), обозначающее количество отправленных байт
recive - целое число без знака (64 бита), обозначающее количество полученных байт (опечатка в названии, должно быть "receive")

#####  class NetworksStaticInfoArray(Structure)
data - указатель на массив структур NetworksStaticInfo
len - размер массива (количество элементов)

###  Управление мониторингом

**`start_monitoring(self, update_interval: float = 2.0)`**
Запускает фоновый мониторинг системы.
Параметры:
update_interval (float, optional): Интервал обновления данных в секундах. По умолчанию 2.0
Действия:
Проверяет, не запущен ли уже мониторинг
Устанавливает флаг _running в True
Сбрасывает событие остановки
Запускает сборщик процессов через DLL
Создает и запускает фоновый поток для регулярного обновления данных

**`stop_monitoring(self)`**
Останавливает фоновый мониторинг системы.
Действия:
- Устанавливает флаг _running в False
- Активирует событие остановки
- Ожидает завершения потока обновления (если он существует)
- Останавливает сборщик процессов через DLL

**`_update_data(self)`**
Внутренний метод для обновления данных системы.
Действия:
- Получает информацию о CPU, памяти и процессах
- Вызывает все зарегистрированные callback-функции с полученными данными
- Логирует ошибки при обновлении данных

###  Работа с callback-функциями

**`register_callback(self, callback)`**
Регистрирует callback-функцию для получения обновлений.
Параметры:
callback: Функция, которая будет вызываться при каждом обновлении данных
Примечание:
Callback-функция должна принимать три аргумента:
- cpu_info - информация о процессоре
- memory_info - информация о памяти
- process_info - информация о процессах

**`unregister_callback(self, callback)`**
Удаляет зарегистрированную callback-функцию.
Параметры:
callback: Функция для удаления из списка callbacks

**`_get_process_info(self) -> List[Dict]`**
Возвращает информацию о процессах.
Возвращает:
Список словарей, каждый содержит:
- pid (str): ID процесса
- name (str): Имя процесса
- cpu_usage (float): Загрузка CPU в %
- memory_mb (float): Используемая память в MB
- read_kb (float): Чтение с диска в KB
- written_kb (float): Запись на диск в KB

**`_get_disk_info(self) -> List[Dict]`**
Возвращает информацию о дисках.
Возвращает:
Список словарей, каждый содержит:
- name (str): Название диска
- total_space (int): Общий объем в байтах
- available_space (int): Доступный объем в байтах

**`_get_network_info(self) -> List[Dict]`**
Возвращает информацию о сетевых интерфейсах.
Возвращает:
Список словарей, каждый содержит:
- name (str): Название интерфейса
- ipv4 (str): IPv4 адрес
- send_speed (float): Скорость отправки в байтах/сек
- recv_speed (float): Скорость приема в байтах/сек

**`get_disk_info(self) -> List[Dict]`**
Публичный интерфейс для получения информации о дисках.

**`get_network_info(self) -> List[Dict]`**
Публичный интерфейс для получения информации о сети.

**`get_cpu_percent(self) -> float`**
Возвращает текущую загрузку CPU в процентах.

**`get_memory_percent(self) -> float`**

**`get_services_info(self) -> List[Dict]`**

Возвращает:
Список словарей, каждый содержит:
- process_id (int): ID процесса сервиса

- status (str): Статус сервиса

###  Управление процессами

##  Модуль Frame.py - GUI

###  Конструктор класса

**`__init__(self, root)`**
Инициализирует графический интерфейс диспетчера задач.
Параметры:
root: Главное окно приложения (tkinter.Tk или tkinter.Toplevel)
Инициализирует:
- Основные настройки окна:
- Заголовок "Диспетчер задач"
- Размер 800x500 пикселей
- Фон темного цвета (#2d2d2d)
Системный монитор:
- Создает экземпляр SystemMonitor
- Настраивает буферы для данных (CPU, память, процессы)
- Добавляет блокировку потоков для безопасного доступа к данным
Интерфейс:
- Настраивает стили элементов
- Создает основной интерфейс с вкладками
- Запускает мониторинг системы с интервалом 2 секунды
- Настраивает обновление GUI каждые 1.5 секунды

###  Методы настройки интерфейса

**`_setup_styles(self)`**
Настраивает визуальные стили для элементов интерфейса.
Настройки включают:
- Темную цветовую схему
- Стили для Notebook (вкладок)
- Стили для Treeview (таблиц)
- Цвета текста и фона

**`_create_interface(self)`**
Создает основной интерфейс приложения с тремя вкладками.
Создает:
- Вкладку "Процессы" (_setup_processes_tab())
- Вкладку "Производительность" (_setup_performance_tab())
- Вкладку "Службы" (_setup_services_tab())
- Использует ttk.Notebook для организации вкладок.

**`_setup_processes_tab(self)`**
Настраивает вкладку с информацией о процессах.
Элементы:
Таблица процессов с колонками:
- ID процесса
- Имя
- Загрузка CPU
- Использование памяти
- Активность диска
- Сетевая активность
- Использование GPU
- Энергопотребление
- Кнопка "Завершить задачу" (_end_task())
- Кнопка "Получить путь" (_get_path())

**`_setup_services_tab(self)`**
Настраивает вкладку с информацией о службах.
Элементы:
- Таблица служб с колонками:
- Имя службы
- ID процесса
- Состояние службы

**`_setup_performance_tab(self)`**
Настраивает вкладку с информацией о производительности.
Элементы:
Canvas для графического отображения метрик
Текстовое поле с текущими показателями
Круговые индикаторы для:
- CPU
- Памяти
- Диска
- Сети
- GPU

###  Методы работы с данными

**`_update_data_buffer(self, cpu_info, memory_info, process_info)`**
Обновляет буфер данных в потокобезопасном режиме.
Параметры:
- cpu_info: Информация о процессоре
- memory_info: Информация о памяти
- process_info: Список процессов
- Использует блокировку (_data_lock) для безопасного доступа из разных потоков.

**`_schedule_gui_update(self)`**
Планирует периодическое обновление интерфейса.
Особенности:
- Вызывает **`_update_gui()`**
- Повторяет вызов каждые 1500 мс (1.5 секунды)
- Использует метод **`after()`** tkinter

**`_update_gui(self)`**
Обновляет все элементы интерфейса на основе данных из буфера.
Вызывает:
- **`_update_processes()`** - обновление таблицы процессов
- **`_update_performance_metrics()`** - обновление показателей производительности
- **`_update_services()`** - обновление таблицы служб
Использует блокировку для безопасного доступа к данным.

###  Методы обновления данных

**`_update_processes(self, processes)`**
Обновляет таблицу процессов.
Параметры:
processes: Список словарей с информацией о процессах
Особенности:
- Сохраняет выделение ранее выбранных процессов
- Форматирует данные для отображения:
- Проценты для CPU
- Мегабайты для памяти
- Преобразует килобайты в мегабайты для диска

**`_update_services(self)`**
Обновляет таблицу служб.
Особенности:
- Получает актуальную информацию о службах через SystemMonitor
- Отображает имя службы, ID процесса и статус

**`_update_performance_metrics(self)`**
Обновляет показатели производительности.
Вычисляет:
- Общее использование дискового пространства
- Общую сетевую активность (исключая виртуальные интерфейсы)
- Использование CPU и памяти
- Форматирует данные для текстового отображения и круговых индикаторов.

**`_update_canvas(self, event=None)`**
Отрисовывает круговые индикаторы на Canvas.
Особенности:
- Автоматически подстраивается под размер Canvas
- Располагает 5 индикаторов в виде "ромашки"
- Использует данные из self.metrics

###  Методы управления процессами

**`_end_task(self)`**
Завершает выбранный процесс.
Логика работы:
- Проверяет, выбран ли процесс
- Запрашивает подтверждение у пользователя
- Вызывает kill_process() из SystemMonitor
- Показывает результат операции

**`_get_path(self)`**
Получает путь к исполняемому файлу процесса.
Логика работы:
- Проверяет, выбран ли процесс
- Получает PID из таблицы
- Вызывает get_proc_path() из SystemMonitor
- Показывает путь в диалоговом окне

###  Деструктор
**`__del__(self)`**
Останавливает мониторинг системы при уничтожении объекта.
Гарантирует:
- Корректное завершение фоновых процессов
- Освобождение ресурсов SystemMonitor
